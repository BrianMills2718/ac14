## Design Decisions (resolved)

**AC14 adapter granularity:** `packet_compilation` stage output is an aggregated dict:
```python
{
    "total_components": N,
    "components_with_empty_rules": N,
    "total_rules": N,
    "total_test_cases": N,
    "components": [{"id": ..., "rules_count": ..., "test_cases_count": ...}, ...]
}
```
Aggregated at top-level for simple criteria expressions; per-component detail in `components` list.

**`codegen` stage output:**
```python
{
    "generation_error": str | None,
    "recomposition_passed": bool,
    "packet_tests_passed": bool | None,
    "failure_category": str | None,
}
```

**`runtime_eval` stage output:**
```python
{
    "total_cases": N,
    "passed_cases": N,
    "failed_cases": N,
    "pass_rate": float,
    "runtime_outputs_passed": bool,
    "cases": [{"case_id": ..., "matched": bool, "mismatches": {...}}, ...]
}
```

**RubricJudge:** Default `StubRubricJudge` (Phase 1). `make trace-eval` passes `--stub` implicitly. Add `--live-llm` flag later.

**Worktree:** Use `worktrees/trace-eval-integration` for Phase 1–5 (AC14 code changes). Merge to master after Phase 5 is verified.
