## Enforced Planning Framework (enforced-planning repo)

The enforced-planning framework (`~/projects/enforced-planning/`) is the governance
layer for this and other AC14-style projects. It enforces three things:

1. **Enforcement gates** — plans cannot be marked Complete without programmatically
   verifiable acceptance criteria. Prevents agents from drifting into wrong metadata.
2. **Read-gating** — source files coupled to ADRs require the linked doc to be read
   before editing. Enforced via `.claude/hooks/gate-edit.sh` (in llm_client).
3. **Doc-code alignment** — plan docs, TODO.md, and CLAUDE.md must be kept in sync
   with code changes. Agents must update all three when a plan completes.

Key files:
- `~/projects/enforced-planning/PLANNING_OPERATING_MODEL.md` — canonical methodology
- `~/projects/enforced-planning/README.md` — framework overview
- `~/projects/ac14/docs/AC14_ANTI_DRIFT_DOCTRINE.md` — AC14-specific anti-drift rules
